Exhibit

Exhibit is a featherweight structured data publishing framework that lets you create web pages with support for sorting, filtering, and rich visualizations by writing only HTML and perhaps some CSS and Javascript code.

It's like Google Maps and Timeline, but for structured data normally published through 3-tier database-backed web sites which can only be afforded by companies and large institutions. Exhibit essentially combines all those 3 tiers into 1 Javascript-based engine and makes it affordable for everyone who has a little bit of knowledge of HTML and small data sets to share with the world.

Why Use Exhibit?

The advantages of Exhibit are as follows:
  • No traditional database technology involved even though Exhibit-embedding web pages appear as if they are backed by databases. So you don't have to design any database, configure it, and maintain it. After all, if you only have a few dozens of things to publish rather than thousands, why would you spend so much effort in dealing with database technologies?
  • No server-side code required even though Exhibit-embedding web pages are heavily templated. So, there is no need to learn ASP, PHP, JSP, CGI, Velocity, etc. There is no need to worry which server-side scripting technology your hosting provider supports.
  • No need for web server if you only want to create exhibits and keep them on your own computer for your own use. They work straight from the file system.

We also provide a complementary service called Babel that lets you convert data from various sources, including tab-separated values copied straight from spreadsheets, into formats that Exhibit understands.

Of course, Exhibit currently only scales to a few hundreds of items, but maybe that's all you need. Furthermore, we are working on optimizing Exhibit and on solutions that let you smoothly transit over to server-side systems as your data sets grow.
